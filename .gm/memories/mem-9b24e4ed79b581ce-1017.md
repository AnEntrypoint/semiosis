{"key":"gm-instruction-emitting-loop-pattern","value":"gm-method lesson (2026-06-29): when a request asks to 'add an LLM-driven self-improving loop' but the user rejects loading a model, the right architecture is an instruction-EMITTING loop: the component emits structured Directive objects (prose instructions) for the calling agent already in the harness to execute, the agent returns Observations, and the refined-then-persisted instruction set IS the trained artifact (instructions-as-weights). This makes a tight VRAM/compute budget trivial by construction since only non-generative pieces run in-process. The user directive 'architect it to instruct the calling agent to do what the llm would do, and the instructions to be the training' is the canonical statement of this pattern. Maps cleanly onto a propose->experiment->observe->refine research loop with explicit failure paths (empty-input early-converge, bounded no-observation, max-cycle cap with converged=False, corrupt-persist fallback-to-defaults)."}
