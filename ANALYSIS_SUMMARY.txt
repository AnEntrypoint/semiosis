================================================================================
KEY HYPOTHESES SUMMARY
================================================================================

HYPOTHESIS 1: Octave Boundaries Should Adapt to Retrieval Performance
  Signal: Per-octave recall@k from evaluation harness
  Action: Refit octaves if recall < 0.3 (suggest contraction) or > 0.9 (coarsen)
  Risk: Medium (requires labeled data, re-fitting expensive)
  Strength: Direct correlation to empirical performance; testable

HYPOTHESIS 2: Entropy Divergence Detects Systematic Retrieval Failures
  Signal: Centroid-member distance entropy, ranking instability
  Action: Trigger consolidate() if coherence < 0.5 or top-k rotation > 30%
  Risk: Low (on-demand, doesn't require labeled data)
  Strength: Early warning; prevents silent failures

HYPOTHESIS 3: Centroid Recomputation Should Trigger on Information Density Shift
  Signal: Node member count growth > 20% or information density drop
  Action: Recompute centroid via _lorentz_mean(), adjust aperture, re-close transitivity
  Risk: Medium (write-heavy, needs batching/amortization)
  Strength: Adaptive; incremental; testable member-to-centroid convergence

HYPOTHESIS 4: Papers Methods Enable Auto-Tuning of Memory Layer Strategies
  Signal: Recall@k under different memory allocation configs
  Action: Bayesian Optimization (or PBT, Hyperband) over memory search space
  Risk: High (requires labeled data, multiple inference runs)
  Payoff: 5-15% recall improvement vs. baseline
